heal observes commit-by-commit metric drift, calibrates Severity from
the codebase's own distribution, and hands the resulting TODO list to
a bundled Claude skill that drains it one finding per commit. This
walkthrough sets all three pieces up.

## Requirements

Before you start.

- macOS or Linux
- A git repository to evaluate
- (Optional) [Claude Code](https://claude.com/code) CLI for the bundled
  `/heal-code-check` and `/heal-code-fix` skills

## 1. Install

Pick one — same `heal` binary regardless of method.

Method-by-method details: [Installation](/heal/installation/).

## 2. Initialize the repository

One command sets up `.heal/`, calibrates, and installs the hook.

```sh
heal init
```

Creates `.heal/`, calibrates from the initial scan, installs the
post-commit hook, and (when `claude` is on your `PATH`) offers to
extract the bundled plugin.

## 3. See the TODO list

The Severity-grouped findings, ready to act on.

```sh
heal check
```

Renders the per-file Severity-grouped TODO list and writes
`.heal/checks/latest.json`. Filters and JSON output:
[CLI › heal check](/heal/cli/#heal-check).

## 4. Audit and drain with Claude

Hand the cache to the bundled skills — one reads, one writes.

```sh
claude /heal-code-check    # read-only architectural audit
claude /heal-code-fix      # drain the cache, one finding per commit
```

Skill contracts: [Claude plugin](/heal/claude-plugin/).
