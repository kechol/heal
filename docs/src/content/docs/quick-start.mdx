**heal turns codebase decay into agent work.** Every commit, it
re-measures complexity, hotspots, and duplication, then hands the
resulting TODO list to Claude — which drains it one fix per commit.
The human steps out of the polling loop. This walkthrough sets that
loop up in three commands.

## Requirements

Before you start.

- macOS or Linux
- A git repository to evaluate
- (Optional) [Claude Code](https://claude.com/code) CLI for the bundled
  `/heal-code-review` and `/heal-code-patch` skills

## 1. Install

Pick one — same `heal` binary regardless of method.

Method-by-method details: [Installation](/heal/installation/).

## 2. Initialize the repository

One command sets up `.heal/`, calibrates, and installs the hook.

```sh
heal init
```

Creates `.heal/`, calibrates from the initial scan, installs the
post-commit hook, and (when `claude` is on your `PATH`) offers to
extract the bundled plugin.

## 3. See the TODO list

The Severity-grouped findings, ready to act on.

```sh
heal status
```

Renders the per-file Severity-grouped TODO list and writes
`.heal/findings/latest.json`. Filters and JSON output:
[CLI › heal status](/heal/cli/#heal-status).

## 4. Audit and drain with Claude

Hand the cache to the bundled skills — one reads, one writes.

```sh
claude /heal-code-review    # read-only architectural audit
claude /heal-code-patch      # drain the cache, one finding per commit
```

Skill contracts: [Claude skills](/heal/claude-skills/).
