**heal はコードベースの劣化をエージェントの仕事に変えます。** コミットのたびに複雑度・Hotspot・重複を再計測し、出てきた TODO リストを Claude に渡して 1 コミット 1 修正で消化します。人間はポーリング役から降りられます。このページではそのループを 3 つのコマンドで立ち上げます。

## 必要環境

始める前に揃えておくもの:

- macOS または Linux
- 評価対象の git リポジトリ
- (任意)[Claude Code](https://claude.com/code) CLI — 同梱の `/heal-code-review` と `/heal-code-patch` スキルを使う場合

## 1. インストール

どの方法でも同じ `heal` バイナリが入ります。好みのものを選んでください。


  </TabItem>
  <TabItem label="Cargo">

```sh
heal --version
```

各方式の詳細は [インストール](/heal/ja/installation/) を参照。

## 2. リポジトリをセットアップ

`.heal/` の作成からフックの設置まで、このコマンド 1 つでまとめて行います。

```sh
heal init
```

初回スキャンの結果から Severity の閾値を決め、post-commit フックをインストールします。`claude` が `PATH` にあれば、同梱スキルセットを `.claude/skills/` 配下に展開するかどうかも確認します。

## 3. TODO リストを見る

Severity ごとにまとまった、すぐ手をつけられる findings 一覧です。

```sh
heal status
```

ファイル単位の Severity 別 TODO リストを表示し、`.heal/findings/latest.json` に書き出します。フィルタや JSON 出力は [CLI › heal status](/heal/ja/cli/#heal-status) を参照。

## 4. Claude に渡して片付けてもらう

同梱スキルにこのキャッシュを渡します。読むだけのスキルと、実際に直すスキルの 2 つを用意しています。

```sh
claude /heal-code-review    # 読むだけ: コード全体を見て改善案を出す
claude /heal-code-patch     # 直す: 1 コミット 1 finding ずつキャッシュを片付ける
```

スキルの詳細は [Code › スキル](/heal/ja/code/skills/) を参照。
