heal はコミットのたびにコードベースのメトリクスを測り、そのコード
ベース自身の分布をもとに Severity の閾値を決めます。出てきた TODO
リストを、同梱の Claude スキルが 1 コミット 1 finding ずつ片付けて
いきます。このページではその一連のセットアップを案内します。

## 必要環境

始める前に揃えておくもの。

- macOS または Linux
- 評価対象の git リポジトリ
- （任意）[Claude Code](https://claude.com/code) CLI — 同梱の
  `/heal-code-check` と `/heal-code-fix` スキルを使う場合

## 1. インストール

どの方法でも同じ `heal` バイナリが入ります。お好みのものを選んで
ください。

<Tabs syncKey="installer">
  <TabItem label="Homebrew">

```sh
brew install kechol/tap/heal-cli
```

  </TabItem>
  <TabItem label="Cargo">

```sh
cargo install heal-cli
```

  </TabItem>
  <TabItem label="Shell">

```sh
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/kechol/heal/releases/latest/download/heal-cli-installer.sh | sh
```

  </TabItem>
</Tabs>

```sh
heal --version
```

各方式の詳細は [Installation](/heal/ja/installation/) を参照。

## 2. リポジトリをセットアップ

`.heal/` の作成からフックの設置まで、このコマンド 1 つでまとめて
行います。

```sh
heal init
```

初回スキャンの結果から Severity の閾値を決め、post-commit フックを
インストールします。`claude` が `PATH` にあれば、同梱プラグインを
展開するかも併せて確認します。

## 3. TODO リストを見る

Severity ごとにまとまった、すぐ手をつけられる findings 一覧です。

```sh
heal check
```

ファイル単位の Severity 別 TODO リストを表示し、
`.heal/checks/latest.json` に書き出します。フィルタや JSON 出力は
[CLI › heal check](/heal/ja/cli/#heal-check) を参照。

## 4. Claude に渡して片付けてもらう

同梱スキルにこのキャッシュを渡します。読むだけのスキルと、実際に
直すスキルの 2 つを用意しています。

```sh
claude /heal-code-check    # 読むだけ: コード全体を見て改善案を出す
claude /heal-code-fix      # 直す: 1 コミット 1 finding ずつキャッシュを片付ける
```

スキルの詳細は [Claude プラグイン](/heal/ja/claude-plugin/) を参照。
