**heal はコードベースの劣化をエージェントの仕事に変えます。** コミットのたびに複雑度・Hotspot・重複を再計測し、出てきた TODO リストを Claude に渡して、1 コミット 1 修正で消化していきます。人間はポーリング役から降りられます。このページではそのループを 3 つのコマンドで立ち上げます。

## 必要環境

始める前に揃えておくもの。

- macOS または Linux
- 評価対象の git リポジトリ
- （任意）[Claude Code](https://claude.com/code) CLI — 同梱の
  `/heal-code-review` と `/heal-code-patch` スキルを使う場合

## 1. インストール

どの方法でも同じ `heal` バイナリが入ります。お好みのものを選んで
ください。

<Tabs syncKey="installer">
  <TabItem label="Homebrew">

```sh
brew install kechol/tap/heal-cli
```

  </TabItem>
  <TabItem label="Cargo">

```sh
cargo install heal-cli
```

  </TabItem>
  <TabItem label="Shell">

```sh
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/kechol/heal/releases/latest/download/heal-cli-installer.sh | sh
```

  </TabItem>
</Tabs>

```sh
heal --version
```

各方式の詳細は [Installation](/heal/ja/installation/) を参照。

## 2. リポジトリをセットアップ

`.heal/` の作成からフックの設置まで、このコマンド 1 つでまとめて
行います。

```sh
heal init
```

初回スキャンの結果から Severity の閾値を決め、post-commit フックを
インストールします。`claude` が `PATH` にあれば、同梱スキルセット
を `.claude/skills/` 配下に展開するかも併せて確認します。

## 3. TODO リストを見る

Severity ごとにまとまった、すぐ手をつけられる findings 一覧です。

```sh
heal status
```

ファイル単位の Severity 別 TODO リストを表示し、
`.heal/findings/latest.json` に書き出します。フィルタや JSON 出力
は [CLI › heal status](/heal/ja/cli/#heal-status) を参照。

## 4. Claude に渡して片付けてもらう

同梱スキルにこのキャッシュを渡します。読むだけのスキルと、実際に
直すスキルの 2 つを用意しています。

```sh
claude /heal-code-review    # 読むだけ: コード全体を見て改善案を出す
claude /heal-code-patch      # 直す: 1 コミット 1 finding ずつキャッシュを片付ける
```

スキルの詳細は [Claude スキル](/heal/ja/claude-skills/) を参照。
