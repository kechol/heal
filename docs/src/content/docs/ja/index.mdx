`heal` はコミットの間に進むコードベースの劣化 — 複雑度の蓄積、Hotspot、重複 — を監視し、見つけた問題を次の Claude セッションが 1 コミット 1 修正で消化していく TODO に変えます。人間がポーリング役を務める必要はありません。

## なぜ heal なのか

AI コーディングエージェントは、依頼された次の変更を作るのは得意です。一方でコードベースは継続的に変化していきます。複雑度が積み重なり、同じファイルが何度も触られ、重複したコードが少しずつ増えていく。エージェントはこうした長期的な変化を観測しません。外部シグナルがなければ、それを追えるのは人間だけです。

`heal` がそのギャップを埋めます。git の post-commit フックがすべての Critical / High finding をコミット出力の中に直接表示し、`heal status` は同じ findings を同梱の `/heal-code-patch` Claude スキルに渡して、1 コミット 1 finding のペースで消化させます。
