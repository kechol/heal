## なぜ heal なのか

AI コーディングエージェントは、依頼された次の変更を生み出すのは得意
です。一方でコードベースは継続的に変化していきます。複雑度が積み重
なり、同じファイルが何度も触られ、重複したコードが少しずつ増えてい
く。エージェントはこうした長期的な変化を観測しません。外部からのシ
グナルがなければ、それを追えるのは人間だけです。

`heal` はそのギャップを埋めます。git の post-commit フックがコミッ
トごとにメトリクスのスナップショットを記録し、Claude の SessionStart
フックが次のセッション開始時にしきい値を超えた変化を提示します。

  <Card title="デフォルトはリードオンリー" icon="approve-check">

  </Card>
  <Card title="単一バイナリ" icon="rocket">
    `cargo install heal-cli` で CLI が入ります。Claude プラグインはバイナリに 同梱されており、`heal
