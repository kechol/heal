import { Card, CardGrid } from '@astrojs/starlight/components';

AI coding agents are great at writing the _next_ thing you ask for.
But your codebase keeps changing on its own — complexity creeps in,
the same file gets touched again and again, copy-paste piles up.
**The agent does not notice. You have to remember.**

HEAL is the bit in the middle: it measures your codebase every commit
and quietly tells your next Claude session when something deserves a
look.

<CardGrid>
  <Card title="Six boring metrics" icon="seti:rust">
    Lines of code, complexity, churn, change coupling, duplication, and a composed hotspot score.
    Computed automatically every commit.
  </Card>
  <Card title="Hook-driven" icon="puzzle">
    A git post-commit hook writes the snapshot. A Claude SessionStart hook surfaces what changed. No
    daemon, no scheduler, nothing to babysit.
  </Card>
  <Card title="Read-only by default" icon="approve-check">
    v0.1 only reads your codebase. Auto-fix lands in v0.2 behind an explicit policy opt-in. Your
    code is safe.
  </Card>
  <Card title="One small binary" icon="rocket">
    `cargo install heal-cli` and you are done. The Claude plugin ships inside the binary; `heal
    skills install` extracts it.
  </Card>
</CardGrid>

:::caution[Status: v0.1]
macOS / Linux only. The CLI and on-disk schemas may shift before
v0.2 lands. Try it, give feedback, expect rough edges.
:::
