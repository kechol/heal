## Why heal

AI coding agents are good at producing the next change you request.
Codebases, however, evolve continuously: complexity accumulates, the
same files get touched repeatedly, and duplicated blocks drift in.
The agent does not observe these long-term shifts — without an
external signal, the human is the only one tracking them.

`heal` fills that gap. A git post-commit hook records a metric
snapshot on every commit; a Claude SessionStart hook surfaces what
crossed a threshold the next time a session opens.

## At a glance

<CardGrid>
  <Card title="Six standard metrics" icon="seti:rust">
    Lines of code, complexity, churn, change coupling, duplication, and a composed hotspot score —
    computed automatically on every commit.
  </Card>
  <Card title="Hook-driven" icon="puzzle">
    A git post-commit hook writes the snapshot. A Claude SessionStart hook surfaces what changed. No
    daemon, no scheduler.
  </Card>
  <Card title="Read-only by default" icon="approve-check">
    By default `heal` only reads the repository. Automated repair is gated behind an explicit policy
    opt-in.
  </Card>
  <Card title="Single binary" icon="rocket">
    `cargo install heal-cli` installs the CLI. The Claude plugin is bundled in the binary and
    extracted by `heal skills install`.
  </Card>
</CardGrid>
